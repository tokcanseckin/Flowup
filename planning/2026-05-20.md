### 3. Critical Bug Fixes — Subscription Launch Blockers

**Auth Headers Fix (1910f7d):**
- **Issue:** `getSong()` API call missing authentication headers
- **Impact:** Premium users couldn't access lyrics even after paying (server returned free-tier response)
- **Fix:** Added `getUserAuthHeaders()` to getSong() call in frontend/src/api/client.ts
- **Root Cause:** When subscription layer was added, getSong() wasn't updated with auth headers like other endpoints

**Datetime Comparison Fix (4fdd707):**
- **Issue:** Comparing offset-naive datetime from PostgreSQL with offset-aware `datetime.now(timezone.utc)`
- **Impact:** `is_subscription_active()` crashed with "can't compare offset-naive and offset-aware datetimes"
- **Fix:** Added `.replace(tzinfo=timezone.utc)` to `user.subscription_expires_at` before comparison
- **Note:** PostgreSQL stores timestamps without timezone info; Python datetime requires explicit timezone awareness

### 4. UX Improvements

**Pricing Page Layout (b7b74fb):**
- Made subscription page more compact for better mobile UX
- Reduced vertical spacing between elements
- Tightened feature list and CTA button positioning

**Playlist Context Persistence (a16906f):**
- **Issue:** Refreshing page on song view lost playlist context (no Next/Previous buttons, Back to Trial button broken)
- **Fix:** Encode `playlist_id` as query param in song URL (`/song/123?playlist_id=5`)
- **Impact:** Users can refresh without losing navigation context, paywall back button works after refresh

**Next-Song Prefetch Optimization (3ce56c0):**
- **Issue:** Prefetch cache key mismatch caused duplicate fetches (prefetch used `undefined` target_lang, actual fetch used user's target_lang)
- **Fix:** Pass `effectiveTargetLang` to prefetch function, check cache before prefetching
- **Performance:** Reduced prefetch delay from 2000ms → 500ms (more responsive autoplay)

**Lookup Panel Spacebar Fix (00c1bad):**
- **Issue:** Pressing spacebar to unpause left lookup panel open (expected: panel closes when resuming playback)
- **Fix:** Close active word/line lookup when spacebar unpauses
- **UX:** Consistent behavior with click-to-unpause (which already closed panel)
