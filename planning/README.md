# Planning Directory

This folder contains weekly plans, daily progress reports, and project documentation for the Flowup/SingoLing project.

## Structure

### Weekly Plans
- **Format:** `week_YYYY-MM-DD.md` (dated by Monday of the week)
- **Content:** High-level goals, daily breakdowns, deferred items, watch-outs, quick wins
- **Purpose:** Strategic overview and prioritization for the week

### Daily Reports
- **Format:** `YYYY-MM-DD.md` (e.g., `2026-05-18.md`)
- **Template:** `daily-template.md` — copy and fill in as needed
- **Content:** Lightweight, flexible tracking — add tasks as you work, capture notes and decisions
- **Purpose:** Daily journal for execution, blockers, and learnings

### Documentation (`docs/`)
Centralized project documentation — architecture, status, ideas, integration guides. See [docs/README.md](docs/README.md) for details.

## Workflow

1. **Monday morning:** Review `week_*.md` to orient on priorities
2. **Each day:** 
   - Open (or create) that day's report file (`YYYY-MM-DD.md`)
   - Add tasks as you start them, check off as you complete
   - Jot down notes, blockers, decisions as they happen
3. **End of day:** Quick review — did anything need to carry forward?
4. **Friday evening:** Review the week, plan next week's `week_*.md`

## Tips

- **No pressure to be exhaustive** — daily reports are for you, not an audit trail
- **Capture blockers immediately** so you can unblock async (e.g., waiting for API keys, design decisions)
- **Use Notes section liberally** — insights that would be hard to rediscover from commit messages alone
- **Copy from daily-template.md** when creating new day files
- **Reference docs/** when clarifying scope or validating approach
